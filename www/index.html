Mission

Generic syntax highlighting support for Intelli/J IDEA. Uses simple configuration files to add syntax highlighting for eg Ruby, Python, etc.

In contrast to writing a full-blown IDEA plugin for every language this plugin should have a new syntax available within minutes. In contrast to the XML filetypes extension that IDEA offers, this plugin is more flexible. (But of course less flexible than writing a full-blown language-specific IDEA plugin.. :)

For now it uses 'regex' based rules to define everything that should be highlighted. You can also youse 'ruby' or 'groovy' to define syntax tokens. This, however, slows down the code highlighting noticeable.

Status

This plugin is usable, but still in an early alpha stage. Major performance issues remaining. Tested only with moderately sized Ruby projects.

Download / Availability

The plugin is available through the IDEA Plugin Manager.

Example Configuration

Here's a part fo the example configuration included in the current release of the plugin: 


[SimpleSyntax:V1.0]

Name: Ruby
Icon: Ruby.png
ExampleCode: Ruby.rb
Description: Ruby Script
(snip)

# Syntax Configuration
ruby  BLOCK_COMMENTS                => tags/BLOCK_COMMENTS.rb
regex LINE_COMMENTS                 => (?m)#.*$
ruby  DOC_COMMENTS                  => tags/DOC_COMMENTS.rb
ruby  SPECIAL_QUOTED_STRINGS        => tags/SPECIAL_QUOTED_STRINGS.rb
regex SINGLE_QUOTED_STRINGS         => \'(?:[^\']|\\')*\'
regex DOUBLE_QUOTED_STRINGS         => \"(?:[^\"]|\\")*\"
ruby  OPERATORS                     => tags/OPERATORS.rb
ruby  KEYWORDS                      => tags/KEYWORDS.rb
ruby  CORE_FUNCTIONS                => tags/CORE_FUNCTIONS.rb
regex CLASS_NAMES                   => [A-Z_]*(?:[A-Z][a-z]+)+
regex CLASS_VARIABLE                => @@[a-z_]+
regex INSTANCE_VARIABLE             => @[a-z_]+
regex SYMBOLS                       => :\w+
regex IDENTIFIERS                   => [a-z]\w*
regex NUMBERS                       => (?:0x[0-9A-Fa-f]+)|(?:[0-9]+(?:\.[0-9]+)?)
regex CONSTANTS                     => [A-Z][A-Z_]*

# Element Descriptions
descriptions[ BLOCK_COMMENTS ] = Block comment
descriptions[ LINE_COMMENTS ] = Line comment
descriptions[ DOC_COMMENTS ] = Documentation
(snip)

# Element Default Attributes
attributes[ BLOCK_COMMENTS ] = #008000
attributes[ LINE_COMMENTS ] = #008000
attributes[ DOC_COMMENTS ] = #008000,BOLD
(snip)

Open Issues

These are the more general issues with the current plugin version:

  • Lexer doesn't cope well with some ways of changing source code. Strange..
  • Highlighting a longer piece of source code can be pretty slow. Especially with lots of external (ruby/groovy) rules..
  • Arranging syntax rules in the right order can be annoying. A simple lexer-like configuration would be cool..
